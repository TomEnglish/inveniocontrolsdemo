Invenio Technologies · ProjectControls

One workspace for project controls — from baseline to closeout.

Material tracking and project visibility for energy and industrial teams. Lock budgets, track field progress, manage change orders, and report earned value on the same data — without the spreadsheet handoffs.

  • Six-role hierarchy — super_admin, admin, pm, pc_reviewer, editor, viewer
  • Project-scoped admins via project membership
  • Earned-value math via Rules of Credit milestones, not guesswork

What it does

Setup

Project shells, COA library, Rules of Credit templates, per-discipline weights, user invites, foreman aliases, project membership.

Execution

Baseline lock, weekly progress imports (CSV/Excel), drawing-level grid with milestone tracking, IWP and foreman filters, period snapshots.

Change management

Three-stage workflow — submit, PC review, PM approve. Hours impact rolls into current and forecast budgets.

Earned value & reports

CPI, SPI, S-curve, period close, variance analysis, snapshot comparison. All sourced from the same canonical progress records.

The pages

A walkthrough of every screen in the app.

Overview

Executive Dashboard

Five live KPIs at the top — overall earned percent, earned hours, CPI, SPI, and a composite percent computed from per-discipline weights. Below: an earned-value-by-discipline chart, an S-curve tracking planned vs earned vs actual hours across reporting periods, and a discipline summary table.

Setup

Project Setup

One project at a time. Disciplines, ROC templates, per-discipline weights, tenant users and invites, foreman aliases. Admins manage projects they're members of; super_admins manage all projects in the tenant.

Setup

COA & Unit Rates

Tenant-wide chart of accounts library. Codes, descriptions, unit-of-measure, base rate, productivity factor adjustment, and effective PF rate. Imports via xlsx; edits flow through an audit-logged RPC.

Setup

Rules of Credit

Per-discipline milestone templates with weights summing to 1.0. Drives the earned-percent calculation across every progress record. Versioned — historical baselines reference the template revision in effect at lock time.

Execution

Budget & Baseline

Three-budget model — Original (locked at baseline), Current (Original + approved COs), Forecast (Current + pending COs). Once locked, discipline budgets are read-only; scope changes flow through Change Management.

Execution

Progress & Earned Value

Drawing-level grid for every progress record on the project. Multi-select chip filters across discipline, IWP, foreman, type, size, spec; free-text line-area search. Eight milestones per record; earned percent computes live.

Execution

Universal Upload

CSV or Excel. Wide alias map covers most field-tracker exports — drawing, description, hours, percent, foreman, IWP, milestones, attribute columns. Uploads create a weekly snapshot for later comparison. Template provided for first-time users.

Execution

Period Snapshots

Frozen captures of project state — first-audit baseline, then weekly. Two radio columns let you pick A and B; the comparison card shows per-record drift in earned percent and earned hours between captures. Green = forward progress, red = regression.

Execution

Change Management

Three-stage CO workflow — editor submits, PC reviewer forwards or rejects, PM approves or rejects. Hours impact rolls into current and forecast budgets immediately. Audit-logged at every transition.

Analytics

Earned Value

Drill-down view. Top KPI band (Earned %, Total Earned, Total Budget, CPI, SPI), per-discipline budget vs earned vs actual chart, and a top-20 records table sorted by earned-hours delta — surfaces the biggest gaps to investigate.

Analytics

Discipline Progress

One card per active discipline. Earned, actual, and CPI stats, plus horizontal bars for earned-vs-budget and actual-vs-budget. Actual-vs-budget turns red when actuals exceed plan.

Analytics

Reports

Cost variance, schedule variance, forecast at completion. CPI/SPI trend chart per locked period. Period Close card to lock the open period — closes the books, freezes BCWP/ACWP, seeds the next period. Variance analysis table per discipline.
